# 人质解救模式 (Hostage Rescue) 适配说明

## 更新概览

我们对系统的 Demo 解析器 (`demoParser`) 以及相关的 UI 呈现 (`Timeline`) 进行了适配，正式支持了 CS2 的人质解救模式 (`cs_` 前缀地图)。

## 适配事件映射表

在处理解析时，我们将人质相关的原生事件集成到了高优先级解析序列中：

1. **`hostage_rescued`** 
   - 描述: CT 玩家成功将人质带回救援区。
   - 解析等级: `5` (与炸弹拆除同级)。
   - 提取信息: 获取救援的玩家 `steamid`，计算胜率变化，并记录为 `hostage_rescued` Timeline 事件。
   - UI 呈现: 蓝色卡片显示「人质登机」，带有 `PersonStanding` 图标。

2. **`hostage_killed`**
   - 描述: 玩家错误击杀人质。
   - 解析等级: `6` (与 C4 爆炸同级，作为回合关键终局节点)。
   - 提取信息: 获取行为玩家 `steamid`。对于友军击杀导致的回合结束有明确的负面记录。
   - UI 呈现: 红色闪烁卡片显示「伤害人质」，带有 `UserMinus` 图标。

## WPA / Rating 3.0 适配

回合胜率引擎（WPA）与 Rating 计分：
- 对局引擎保留原有的事件奖励惩罚机制。虽然当前未直接为 `hostage_rescued` 事件配置超额的 WPA 或 Rating 分数（默认为回合胜率贡献），但回合终止带来的事件结算已自动计入最后存活状态和队伍阵营胜负结算（`round_end`）。 
- T 阵营和 CT 阵营的数据模型在防守与进攻职能上有所翻转，但基础的击杀/贸易击杀逻辑保持高度兼容。

## UI 优化详情

- **TimelineBetaTab**: `hostage_rescued` 与击杀、下包、拆包等被列为核心里程碑事件，在「简化模式」的轴线上会默认展示该事件节点。
- 对人质事件追加了独立的专属色彩主题 (蓝色表示获救，红色表示受害)，增强复盘时对这类特殊比赛关键时间点的高效识别。
